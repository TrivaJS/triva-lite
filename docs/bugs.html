Reporting Bugs

Triva is used in real-world production environments, and correctness matters. Bug reports are treated as first-class contributions.

What to Report

  • Incorrect throttling behavior
  • Unexpected rate limit enforcement
  • Parser inaccuracies
  • Log inconsistencies or corruption
  • Performance regressions

How to Report

Please include:

  • Node.js version
  • Framework (Express, Fastify, native)
  • Configuration used
  • Minimal reproduction if possible

Call to Action

If you encounter unexpected behavior, open an issue on GitHub. Clear bug reports directly improve the reliability of the ecosystem for everyone.

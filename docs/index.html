Triva Overview

Triva is a developer-centric toolkit designed to provide robust request/event logging and intelligent middleware controls for modern web applications. It is built with enterprise and OSS standards in mind, supporting SOC2 / ISO-compliant principles.

Package Components

The Triva package provides two core resources for developers:

  • Request/Event Logs: Locally cached, structured logs capturing every request, middleware decision, and environment metadata. Fully queryable and filterable for monitoring, analytics, and compliance.
  • Middleware: Includes behavior-aware Throttle, request Parser, and Rate Limiter. Controls request flow, protects APIs, and attaches metadata to logs.

Use Cases

Triva is ideal for:

  • Public APIs: Protect against scraping, credential stuffing, and accidental overload.
  • Enterprise Dashboards: Handle bursty UI requests and refresh patterns without blocking legitimate users.
  • Mixed Human/Bot traffic: Differentially rate-limit known bots and AI crawlers while maintaining fair access for users.
  • Tiered Applications: Configure policies per account level for free, paid, or enterprise clients.

Getting Started

To begin using Triva, install the package, integrate middleware, and optionally configure the logs system. See Install for detailed setup instructions.

Developer Philosophy

Triva emphasizes determinism, transparency, and storage-agnostic operation. It never relies on timers or background jobs. All request decisions are attached to req.triva for inspection and auditing.

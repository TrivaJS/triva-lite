Installation

Triva is distributed as a standard Node.js package and is designed to integrate cleanly into existing HTTP servers and frameworks. There is no global initialization step, no background services, and no required external dependencies.

Installing the Package

Install Triva using your package manager of choice: 

npm install triva
# or
yarn add triva

Importing Triva

Triva exposes a small, intentional API surface. Most applications will only need the middleware factory and the logs interface. 

import { createMiddleware, logs } from 'triva';

Creating the Middleware

All configuration happens at middleware creation time. This includes throttling, parsing, rate limiting, and logging behavior. Once created, the middleware is a pure function that can be attached to any compatible HTTP server. 

const middleware = createMiddleware({
    throttle: {
        limit: 1500,
        window_ms: 24 * 60 * 60 * 1000,
        burst_limit: 25,
        burst_window_ms: 1000
    },
    parser: true
});

What Happens After Installation

Once attached, Triva automatically:

  • Evaluates request behavior for throttling and rate limiting
  • Parses request metadata (UA, engine, OS, device, bot/AI indicators)
  • Generates structured request and event logs
  • Attaches decision metadata to req.triva

No additional setup is required to begin logging or enforcing limits.
